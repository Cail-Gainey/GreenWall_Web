# GreenWall

GreenWall 是一款用于创作 GitHub 风格贡献图的工具。用户可以在日历网格上绘制文字、数字和图案，将作品保存、导入、导出、发布到社区，也可以在完成 GitHub 授权后，把图案转换为仓库提交记录，生成个性化的贡献图效果。

本仓库用于发布 GreenWall 的版本说明和使用文档。请在 Releases 页面下载最新版本，或访问在线版本使用。

## 主要功能

- 贡献图绘制：按年份绘制 GitHub 风格贡献日历，支持不同贡献强度颜色。
- 图案模板：内置字母、数字等模板，也可以使用收藏或导入的模板。
- 社区分享：浏览公开作品，支持点赞、收藏、评论、导入和导出。
- GitHub 推送：连接 GitHub 后，将绘制结果推送到新仓库或已有仓库。
- 个人中心：管理个人资料、头像、个人作品、收藏作品和关注关系。
- 管理后台：管理员可维护用户、角色、权限、公告、参数、字典、日志和系统状态。

## 使用指南

### 1. 登录或注册账号

打开 GreenWall 后，可以先浏览首页和公开社区内容。发布作品、收藏作品、评论、进入个人中心或使用 GitHub 推送功能时，需要登录账号。

点击页面右上角的登录入口，输入账号、密码和验证码即可登录。没有账号时，可以切换到注册页面，根据提示填写账号、邮箱、密码和验证码完成注册。忘记密码时，可以在登录弹窗中切换到找回密码页面，通过邮箱验证码重置密码。

### 2. 绘制贡献图

进入首页后，选择要绘制的年份。页面会生成对应年份的贡献日历网格，每个格子代表一天。选择画笔和贡献强度后，在网格上点击或拖动即可绘制图案。

如果画错了，可以使用橡皮擦清除单个格子，也可以使用清空功能重新开始。绘制完成后，可以继续编辑、导出图片、导出数据、上传社区或进入 GitHub 推送流程。

### 3. 使用图案模板

点击首页中的图案或模板入口，可以打开模板选择窗口。选择字母、数字或收藏模板后，再回到贡献图网格中放置模板。

模板适合快速绘制文字、数字和常用图案。使用模板后仍然可以继续用画笔微调细节。

### 4. 发布到图案社区

绘制完成后，可以将作品发布到社区。发布前需要填写标题、描述和可见范围。公开作品会出现在社区列表中，其他用户可以浏览、点赞、收藏、评论或导入。

如果当前画布为空，系统会阻止发布。发布前建议确认图案内容完整，并选择合适的可见范围。

### 5. 浏览和互动社区作品

进入社区页面后，可以查看其他用户发布的贡献图作品。点击作品卡片可打开详情，查看完整图案、作者、描述、评论和互动数据。

登录后可以对作品点赞、收藏、评论，也可以将作品导入到自己的首页画布继续编辑。导入作品通常会覆盖当前画布，导入前请确认当前作品是否已经保存或导出。

### 6. 连接 GitHub 并推送贡献图

在首页完成绘制后，可以点击 GitHub 相关入口进行授权。授权成功后，GreenWall 会返回系统并显示 GitHub 连接状态。

推送时可以选择创建新仓库，也可以选择已有仓库。提交任务后，系统会在后台生成对应日期的提交记录，并展示任务状态。任务成功后，可以前往 GitHub 查看贡献图效果。

推送失败通常与 GitHub 授权、仓库权限、仓库名称、网络状态或当前画布为空有关。请根据页面提示检查后重新操作。

### 7. 管理个人作品和收藏

登录后可以进入个人主页查看自己的资料和作品。自己的作品支持编辑、删除、导入和导出；其他用户访问个人主页时，只能看到有权限查看的作品。

进入我的收藏页面后，可以查看已收藏的作品。收藏作品可以取消收藏，也可以导入到首页画布继续使用。

## 常见问题

### 为什么不能发布作品？

常见原因包括未登录、当前画布为空、没有发布权限、标题或可见范围填写不完整。请先登录账号，确认贡献图中至少有一个有效格子，再重新提交。

### 为什么 GitHub 推送失败？

可能是尚未连接 GitHub、授权已过期、仓库名称不符合要求、已有仓库没有写入权限、当前画布为空，或 GitHub 网络访问异常。请根据任务失败提示检查原因。

### 导入社区作品会保留当前画布吗？

导入作品通常会覆盖当前首页画布。如果需要保留当前内容，请先导出图片或数据，再导入新作品。

### 验证码或注册失败怎么办？

请确认验证码未过期、邮箱格式正确、密码符合页面规则，并检查是否已勾选隐私条款。如果仍然失败，可以稍后重试或联系管理员。

## 数据与隐私

GreenWall 会根据功能需要保存账号信息、作品内容、收藏记录、评论记录、GitHub 授权状态和系统操作记录。GitHub 推送需要用户主动授权，系统只会在授权范围内执行相关操作。

请不要在作品标题、描述、评论或公开截图中填写密码、令牌、私钥等敏感信息。
